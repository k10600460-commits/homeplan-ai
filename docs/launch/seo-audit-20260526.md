## 2. 基本 SEO 要素

`src/app/layout.tsx:11-14`（グローバル設定）:
```
title:       "SplanAI — AI Floor Plan Generator for Home Builders"
description: "Turn any lot into 3 custom floor plan proposals in 30 seconds. AI-powered

```
- 文字数: title 約52文字 ✅（上限60文字以内）
- description 約150文字 ✅（上限160文字以内）
- **問題**: `layout.tsx` にグローバル設定のみ。`page.tsx` は `"use client"` のためページレベル metadata をエクスポートできない。現状は layout の設定がそのままLP（`/`）に適用されており実害はないが、将来ページが増えた場合に canonical/description の設定漏れリスクあり。

### 2-2. sitemap.xml

- `/login` に SEO 価値はなく不要（ただし有害でもない）

- ただし将来コンテンツページ（ブログ等）を追加した場合は要更新


これらのページはインデックス不要なので実害はほぼないが、技術的には誤り。

## 4. 追加確認事項

### 4-1. `lang` 属性とスペイン語コンテンツ

`src/app/layout.tsx:40`:
```html
<html lang="en">
```
LP はスペイン語コンテンツを JS で切り替える（同一 URL `/`）。  
`hreflang` タグは未実装。Google は同一 URL の JS 言語切り替えを別ページとして扱わないため、スペイン語版のインデックス機会はない（軽微）。

### 4-2. X-Robots-Tag ヘッダー

`next.config.ts` の `securityHeaders` に `X-Robots-Tag` は含まれていない ✅（クロール阻害なし）

### 4-3. `"use client"` の SEO 影響

LP (`page.tsx:1`) は `"use client"` → クライアントコンポーネント。  
Next.js App Router ではクライアントコンポーネントも SSR でサーバーレンダリングされるため、Googlebot は HTML を受け取れる。ただし `useState` の初期値でレンダリングされるため、言語切り替え後のコンテンツは JS 実行まで見えない。  
実害: ほぼなし（デフォルト英語コンテンツは SSR で出力される）

### 🟡 中（SEO 機会損失 / 技術的誤り）

#### M-1: 構造化データ（JSON-LD）の完全欠如
- **根拠**: src/ 全体に `application/ld+json` なし
- **影響**: FAQ リッチリザルト・SoftwareApplication カード取得不可。競合が実装している場合に SERP で見劣りする。
- **修正案**:
  ```typescript
  // layout.tsx または LP の <head> に追加
  <script type="application/ld+json">{JSON.stringify({
    "@context": "https://schema.org",
    "@type": "SoftwareApplication",
    "name": "SplanAI",
    "applicationCategory": "BusinessApplication",
    "offers": { "@type": "Offer", "price": "0", "priceCurrency": "USD" },
    "url": "https://splanai.com"
  })}</script>
  ```
  FAQPage スキーマ（`page.tsx:93-99` の 5 FAQ 項目）も追加推奨。

#### M-2: グローバル canonical が全ページに伝播
- **根拠**: `layout.tsx:16` — `alternates: { canonical: "https://splanai.com" }`
- **影響**: `/login`・`/upgrade` 等が canonical: `https://splanai.com` を出力する（技術的誤り）
- **修正案**: layout の canonical を削除し、LP の layout（または LP 専用の metadata）にのみ canonical を設定する。または各ページが `metadata` をエクスポートして canonical を個別指定する。

#### M-3: sitemap に SEO 価値のないページが含まれる
- **根拠**: `sitemap.ts:7` — `/login` が `priority: 0.6` で登録
- **影響**: 小さいが Search Console でインデックス対象として処理されるリソースの無駄
- **修正案**: `/login` をサイトマップから除外する。

### 🟢 軽微

#### L-1: H1 にメインキーワードが含まれない
- **根拠**: `page.tsx:552-554` — H1: "Show clients their dream home before they sign."
- "AI floor plan" "home builder" 等のターゲットキーワードが H1 にない
- **修正案**: サブタイトル行や H1 のサブスパンにキーワードを組み込む（例: "AI Floor Plans for Home Builders"）

#### L-2: `twitter:site` タグなし
- **根拠**: `layout.tsx:26-31` — twitter オブジェクトに `site` プロパティなし
- **修正案**: Xアカウントが確定したら `twitter: { site: "@splanai" }` を追加

#### L-3: `og:image:alt` なし
- **根拠**: `layout.tsx:24` — `images: [{ url: "...", width: 1200, height: 630, alt: "..." }]`
  → 実際には alt は設定済み (`"SplanAI — AI Floor Plans for Home Builders"`) ✅
  → 問題なし（調査で確認済み）

#### L-4: 画像コンテンツが皆無
- LP のビジュアルは絵文字・CSSグラデーション・SVGのみ。Image Search 流入機会なし。
- **修正案**: デモGIF（OI-004）や実際のプラン画像を追加すると alt テキストで SEO 貢献も得られる。

#### L-5: スペイン語 hreflang なし
- 同一 URL で言語切り替えのため構造的に対応困難。
- **修正案**: `/es` サブパスを用意するか、現状維持でスペイン語 SEO は諦める（Phase 0 では問題なし）。
